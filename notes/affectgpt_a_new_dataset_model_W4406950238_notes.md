## 📚 Reading Notes
 
### 1. The Core Problem
*   **Data Scarcity**: Lack of large-scale datasets with **intensive, descriptive emotion annotations** necessary to unlock natural language describing capabilities of MLLMs.
*   **Fusion Deficit**: Mainstream Audio-Video LLMs (AV-LLMs) leave all cross-modal interactions to the LLM decoder, failing to emphasize multimodal characteristics crucial for Multimodal Emotion Recognition (MER).
*   **Benchmark GAP**: Traditional metrics (Accuracy/F1) struggle with the free-form, open-ended natural language responses generated by MLLMs.
 
### 2. Architecture & Methodology
*   **AffectGPT Model**: Introduces a **Pre-fusion Operation** that moves cross-modal interaction *outside* the language model to enhance integration.
*   **Two Pre-fusion Variants**:
    1.  **Q-Former based**: Preserves temporal information; uses learnable query tokens that interact with concatenated audio-video embeddings via cross-attention.
    2.  **Attention based**: Direct temporal average pooling followed by dynamic weight allocation to emphasize top modality importance.
*   **Backbones**: Uses CLIP ViT-L (Vision), HUBERT-L (Acoustic), and **Qwen-2.5** (LLM) utilizing LoRA adapters ($r=16$) for efficient fine-tuning.
 
### 3. Data & Implementation
-   **MER-Caption Dataset**: Largest descriptive emotion dataset containing **115K coarse-labeled** and **31K fine-labeled** samples with open-vocabulary fine-grained tags.
-   **Model-Led Human-Assisted Annotation**:
    1.  **Generation**: Integrates Audio prompts (SALMONN) and Visual prompts (Chat-UniVi) via GPT-3.5 reasoning headers.
    2.  **Low-Level Filtering**: Removes mismatched speaker triggers (using TalkNet) and length anomalies.
    3.  **High-Level Filtering (Model-Based Crowdsourcing)**: Matches generated descriptions against consensus predictions from multiple traditional discriminative classifiers.
-   **Training**: Batch size 3, AdamW, learning rate 1e-5, trained on 80GB A100.
-   **Code Available**: Yes - [GitHub link](https://github.com/zeroQiaoba/AffectGPT)
 
### 4. Key Results & Takeaways
-   **Unified Performance**: AffectGPT achieves >9% gains on general scores against single-stage integrations (Emotion-LLaMA, PandaGPT) across basic, fine-grained, and sentiment splits.
-   **MER-UniBench**: Validates metric structures by resolving set-level synonyms and layout counts for free-form predictions.
-   **Pre-fusion validation**: Separate Q-former/Attention layers yield better scores than forcing the main text LLM layer to perform holistic temporal binding.
 
### 5. Personal Application
-   **Scaling Strategy**: The *Model-Led Human-Assisted* loop serves as a blueprint for gathering soft labels on unannotated video datasets without manual verification budgets.
-   **Cross-Modal Conflict mitigation**: Explicit project integration (Pre-fusion) might prevent the modality description text from merging too early and causing conflicting weights in dense CoT reasoning layers.
