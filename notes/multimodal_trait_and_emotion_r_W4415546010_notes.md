# 📝 Multimodal Trait and Emotion Recognition via Agentic AI

> [!ABSTRACT] TL;DR
> This paper presents a modular Agentic AI system for multimodal emotion and Big Five personality trait recognition. By orchestrating specific language model calls (a Perception Agent, an Inference Agent, and a Dialogue Agent) and grounding them with contextual metadata (response time, body language annotations, speech features), the system demonstrates that agent-based chain-of-thought (CoT) prompting achieves highly empathetic dialogue.

## 📚 Reading Notes

### 1. Core Objectives
- Most conversational systems respond using static generation based purely on text, ignoring the subtle behavioral or psychological states of the user. 
- The objective was to create an end-to-end framework capable of dynamically inferring emotions and personality (Big Five), and adjusting tone and content generated by a Dialogue Agent.

### 2. Methodological Approach (Agentic Loop)
The system operates on an `Observe → Reflect → Act → Self-Audit` agentic loop using two distinct LLM backbones (LLaMA 3.2 1B vs Falcon-RW-1B) for benchmarking.

#### (A) Perception Agent (Emotion)
- Parses the user’s text *alongside* metadata (Response time: Fast/Slow, Body Language: Relaxed/Fidgeting, Speech Features: Loudness/Hesitancy).
- **LLaMA Config**: Uses a deterministic CoT prompt to output a single categorical emotion label.
- **Falcon Config**: Uses few-shot prompt templates (4-5 examples) to prevent hallucinations, defaulting to keyword-matching heuristic fallbacks if ambiguous.

#### (B) Inference Agent (Personality)
- Estimates continuous traits (Openness, Conscientiousness, Extraversion, Agreeableness, Neuroticism) from `0.0` to `1.0`. 
- **LLaMA Config**: Uses few-shot explicit JSON schema prompting. Maps rules directly into the prompt (e.g., "If Friendly greetings: Extraversion 0.6-0.8").
- **Falcon Config**: Employs a lightweight, non-generative rule-based heuristic mapping the *Perception Agent's* emotion string directly to predefined trait values to save compute/latency.

#### (C) Retrieval Memory & Dialogue Agent
- **Memory**: A TF-IDF vectorizer over text + one-hot emotion + numerical trait embedding retrieves the top-k past cases (either static benchmarks or dynamic prior turns).
- **Dialogue Agent**: Generates the final output. LLaMA shifts its empathetic generation based on the traits (e.g., highly neurotic gets calmer tones). Falcon pulls from exact template banks.

### 3. Key Findings & Trade-offs
- **LLaMA (Generative/Agentic)**: Produced highly diverse (Distinct-2: 0.958), lengthy (88.7 words avg), and deeply empathetic responses. High computational latency (2.07s).
- **Falcon (Template/Heuristic)**: Extremely low latency (0.08s), brief responses (11.6 words avg). 
- **Consistency**: Most impressively, both the complex JSON CoT prompt (LLaMA) and the simple heuristic map (Falcon) achieved overlapping psychological profiles on identical users, sporting a Mean Absolute Error (MAE) difference on trait values of only `0.164`.

### 4. Application Strategy
-   **Structured JSON Output for Soft Labels**: The strategy of enforcing JSON schema templates in the LLM System Prompt to return continuous `0.0` to `1.0` "soft label" variables for personality traits is an excellent architecture for integrating prompt-based extraction into downstream classical logic or tracking nodes.
-   **Metadata Injection**: Explicitly mapping non-textual cues (e.g. `Response time: {rt}, Body language: {bl}`) as formatted f-strings inside a text prompt allows language models to confidently reason across modalities without requiring a multi-modal vision-language architecture.
